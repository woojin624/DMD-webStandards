2주차 수업 복습 내용
  1. XHTML
1. XHTML = HTML + XML
HTML - 순차적 마크업
  1. 한 문서에 기본 데이터, 구조, 표현 정보가 저장
  2. 어떻게 시각적으로 표현할 것인가
  3. 폰트, 색상, 여백, 줄 간격 등 표현에 관련된 다수의 추가 정보
  4. HTML이 대표적인 '순차적 마크업'
XML - 서술적 마크업
  1. 기본 데이터와 구조(의미)만 가지는 문서
  2. 문서 구조와 표현 정보를 분리
  3. 이질적인 환경에서 데이터 교환 형태로도 적합
  4. XML이 대표적인 서술적 마크업

그래서 XHTML은 순차적 마크업의 기능 + 서술적 마크업의 장점을 가져와서
HTML을 XML로 재구축한 마크업 언어

2. XHTML의 서식 #1
  1. 종료 태그 생략 불가능
  2. 소문자 사용
  3. 종료 태그가 없는 경우 <TAG />형식으로
  4. 모든 속성값에 인용 부호 ("")사용
2. XHTML의 서식 #2
  1. 속성에 속성값 생략 불가능
  2. 잘못된 중첩 사용 불가능
  3. <,>,&를 <, >, &로 변환하여 사용
3. XHTML의 구성요소
  1. 태그(tag) - <와 >로 묶어서 표현하는 명령어. 시작,종료 태그(빈 요소는 ).
  2. 요소(element) - 시작 태그부터 종료 태그까지의 모든 명령어 집합.
  3. 속성(attribute) - 태그의 의미와 필요에 따라 가지는 개별적인 옵션. 공백으로 구분.
  4. 값(value) - 속성이 가지는 값
4. XHTML문서의 기본 구조
  1. 1. - 문서형 정의 선언. 엄격형(Strict), 호환형(Transitional)등
  2. 2. - 웹 페이지의 시작과 종료 / 머리 부분과 본문 부분
  3. 3. - 머리 부분 / 일반적인 정보, 문서의 제목 등을 선언
  4. 4. - 본문 부분 / 웹 브라우저 화면에 출력
DTD / xmlns / lang 1. 문서형 정의 및 선언 (DTD : Document Type Definition)
  1. 마크업 문서 작성을 위해 문서의 첫 머리에 선언
  2. 요소와 속성 등을 어떤 규칙에 따라 기술해야 하는지의 기준
  3. 브라우저 렌더링의 기준
  4. 특정 요소의 사용이 제한되거나 규칙이 다르게 적용
  5. 크로스 브라우징 환경에 반드시 필요
엄격형. Strict

- 구조와 표현을 분리하기 위해 단계적으로 사라질 ‘표현’에 관현 요소와 속성을 배제한 엄격한 문서 타입

– Center, font, iframe, strike, u 등의 요소와 target 속성이 제한

호환형. Transitional

– 예전 문서와 strict DTD 중간 단계로 설정한 문서 타입

2. 네임 스페이스 (xmlns)

- XHTML은 HTML을 XML로 재정의한 언어이기 때문에 XML이 가지는 확장성을 이용하여 다른 XML 형식의 문서 표준을 네임스페이스(xmlns)를 이용하여 지원

- HTML을 XML로 재정의한 언어인 XHTML은 위과 같이 정의

3. 휴먼 랭귀지 (lang)

- 한국어, 영어 또는 독일어 등의 인간이 사용하는 언어를 HTML 문서에 선언, (x)HTML을 해석할 수 있는 소프트웨어들이 올바르게 동작하도록 유도

-/p>• 주요 사용 언어에 따라‘한국어(ko), 영어(en), 일어(ja), 중국어(zh), 프랑스어(fr) ...’형식으로 지정

XHTML 요소 1. 문자 코드 세트와 문서 정보 (meta) • 문자 코드 세트 지정하기 – utf-8 : 유니코드. 전세계의 모든 문자를 표현. 국제적인 코드 규약 – euc-kr : 2,350개의 한글문자, 한국에서 통용되는 한자, 영문. • 문서 정보 - 키워드 – 웹 페이지 홍보의 수단으로 검색 키워드 지정 • 문서 정보 - 그 외 정보 – 웹 페이지 문서 내부의 정보를 작성 • 웹 문서의 제목을 선언 – 문서마다 유일한 내용으로 구성 – Title요소는 문서를 구별할 수 있는 첫 번째 관문 – 즐겨찾기 및 북마크에 해당 웹 문서를 추가할 경우에 사용되는 북마크 이름 2. 문서 제목 • 웹 문서의 제목을 선언 – 문서마다 유일한 내용으로 구성 – Title요소는 문서를 구별할 수 있는 첫 번째 관문 – 즐겨찾기 및 북마크에 해당 웹 문서를 추가할 경우에 사용되는 북마크 이름 • 페이지와 웹 사이트 정보 함께 표기하기 – ‘웹 표준 퍼블리싱’ 웹사이트의 ‘묻고 답하기’ 페이지 • 웹 사이트 명이 먼저? 페이지 명이 먼저? – 시각적으로는 웹사이트 명이 먼저. – 음성에 의해 정보를 순차적으로 제공한다면? • 의미 없는 장식 문자 금지 – 스크린 리더와 같은 보조기기를 통해 접근 할 경우 3. 블록 요소와 인라인 요소 • 블록 요소 (Block element) – 독립적인 형태의 상자 – H1~h6, p, div – 인라인 요소, 텍스트, 또 다른 블록 레벨 요소를 포함 • 인라인 요소 (inline element) – 블록 상자 안의 일부 – a, img, em, strong, span – 블록 요소를 포함할 수 없다 – 반드시 블록 레벨 요소에 포함시켜 표현
문단 관련 요소 제목 <h1>~<h1>
  • 대제목, 중제목, 소제목 등
  • h1, h2, h3, h4, h5, h6
  • 잘 구성된 제목은 기기에서의 정보 인식성에 효과적
  • 디자인상 생략된 헤딩 요소를 콘텐츠의 특성에 맞게 마크업할 때 작성하면 순차적으로 정보에 접근하는 시각장애인에게 도움이 된다.
  • 화면에 헤딩 요소가 표시되어 디자인상 문제가 되는 부분은 CSS를 이용하여 숨길 수 있다.
